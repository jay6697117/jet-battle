# Jet Battle — 现代喷气机空战游戏设计规格

## 1. 游戏概述

**Jet Battle** 是一款基于 Three.js 的现代喷气机 3D 空战游戏，同时支持桌面和移动端。玩法定位为"半拟真格斗 + 简化操控"——战斗系统有深度（武器热度、导弹锁定、能量管理），但飞行操控足够简单直觉，让任何人 30 秒内上手。

### 核心设计理念

| 维度 | 目标 |
|---|---|
| **操控** | 简化键盘式：WASD 控制俯仰+转弯，↑/↓ 控制油门，Shift 加速，自动稳定 |
| **爽感** | 锁定导弹追踪、爆炸粒子、击杀特写、音效反馈 |
| **策略** | 武器热度管理、导弹有限、能量管理（速度↔高度）、雷达意识 |

## 2. 操控系统

### 2.1 桌面端（键盘 + 鼠标）

```
┌──────────────────────────────────┐
│  【方向操控】                     │
│        ↑ (W) 爬升                │
│   ← (A) 左转    → (D) 右转      │
│        ↓ (S) 俯冲                │
│                                  │
│  【油门控制】                     │
│   ↑ 方向键  = 加速（油门 +1 档） │
│   ↓ 方向键  = 减速（油门 -1 档） │
│   Shift     = Boost 加速 (限时)  │
│                                  │
│  【战斗操作】                     │
│   Space   = 机枪射击             │
│   E       = 发射导弹             │
│   Q       = 释放干扰弹           │
│                                  │
│  【辅助功能】                     │
│   F       = 自动稳定（回到水平）  │
│   Tab     = 排行榜               │
│   Esc     = 设置菜单             │
│                                  │
│  【视角与锁定】                   │
│   鼠标    = 自由视角旋转         │
│   鼠标右键 = 锁定目标            │
└──────────────────────────────────┘
```

**操控模型**：
- 飞机始终沿机头方向飞行
- `W/S` 控制俯仰（抬头/低头），飞机会自然爬升或俯冲
- `A/D` 控制转弯（内部实现为偏航 + 自动倾斜，视觉效果好看但操控简单）
- 按住 `A/D` 时飞机自动倾斜，松手自动回正
- `F` 一键自动回到水平飞行，新手友好

**油门与速度模型**：
- 油门分 5 档：`0%` → `25%` → `50%` → `75%` → `100%`
- `↑` 方向键按一次油门升一档，`↓` 方向键按一次降一档
- 飞机的巡航速度由油门档位决定，飞机会自动加减速到目标速度
- 起飞默认油门 50%，玩家根据战况随时调整
- `Shift` 按住触发 Boost：在当前速度基础上再加 50%，持续最多 3 秒，消耗 Boost 能量条
- Boost 能量条耗尽后需 5 秒自然恢复
- 速度影响机动性：低速时转弯更灵活，高速时转弯半径更大
- 极低速（油门 0% 且无动能）会触发失速警告，飞机自动缓慢下坠

### 2.2 移动端（虚拟控件）

```
┌──────────────────────────────────────┐
│                                      │
│   ┌───┐                    [导弹]    │
│   │   │ 虚拟摇杆           [干扰弹]  │
│   │ ◉ │ (左手)                       │
│   │   │ 上下=俯仰                    │
│   └───┘ 左右=转弯       ┌────────┐  │
│                          │  FIRE  │  │
│   [BOOST]                │  开火  │  │
│                          └────────┘  │
│         ═══╪═══                      │
│         油门滑杆                      │
│         (底部居中)                    │
└──────────────────────────────────────┘
```

- **左侧虚拟摇杆**：控制俯仰 + 转弯（和 WASD 对应）
- **右侧 FIRE 按钮**：机枪射击（按住连射）
- **右上角功能键**：导弹发射、干扰弹
- **底部油门滑杆**：上推加速，下推减速
- **BOOST 按钮**：左下角，按下短暂超速
- **自由视角**：单指在空白区域滑动旋转相机
- **锁定目标**：双击敌机方向

## 3. 战斗系统

### 3.1 武器系统

| 武器 | 弹药 | 特点 |
|---|---|---|
| **机枪** | 无限，热度限制 | 持续射击积热，过热冷却 3 秒，鼓励点射 |
| **导弹** | 每局 6 发 | 需锁定 2 秒，可被干扰弹规避，击中高伤害 |
| **干扰弹** | 每局 4 发 | 规避来袭导弹，按 Q 释放 |

### 3.2 武器热度系统

```
热度值: 0% ──────────────── 100%
         ░░░░░░░░░░░░░░░░░░
         安全    警告    过热
         0-60%   60-80%  80-100%

- 每秒射击增加 ~8% 热度
- 停止射击每秒降低 ~5% 热度
- 超过 100% 过热锁定，需要冷却 3 秒才能再射击
- HUD 上热度条：绿色 → 黄色 → 红色
```

### 3.3 导弹锁定系统

```
锁定流程:
1. 敌机进入视野 → 出现菱形框
2. 按住右键/双击 → 锁定环开始收缩（2 秒）
3. 锁定完成 → "LOCK" 提示音 + 锁定框变红
4. 按 E 发射 → 导弹追踪飞向目标
5. 目标可释放干扰弹规避

锁定条件:
- 目标在视野前方 60° 锥角内
- 距离 < 2000m
- 无障碍物遮挡
```

### 3.4 能量管理（策略核心）

```
高度 ↑        爬升消耗速度
  |          ╱
  |        ╱   速度↔高度 可互换
  |      ╱
  |    ╱
  ──────────→ 速度 →

- 爬升时速度降低（势能增加）
- 俯冲时速度增快（势能转动能）
- Boost 加速消耗 Boost 能量条，渐恢复
- 低速时机动性下降（转弯半径变大）
- 极低速触发失速警告
```

## 5. 游戏模式

### 5.1 MVP — 自由空战（Solo Mission）

- 玩家 vs AI 敌机
- 按 `E` 生成一波敌机（5 架）
- AI 敌机有简单的追踪和攻击行为
- 被击落后按 `R` 重生
- 排行榜记录 Kills / Deaths / K/D / 存活时间

### 5.2 后续扩展

- **PvP 竞技场**：多人在线对战（WebSocket）
- **任务模式**：保护目标、拦截轰炸机、穿越检查点
- **Boss 战**：超大型敌机/空中堡垒

## 6. 视觉与音效

### 6.1 视觉效果

- **天空盒**：动态云层 + 太阳光照
- **飞机模型**：低多边形现代战斗机（F-16 风格），带尾焰粒子
- **粒子效果**：
  - 机枪射击火花
  - 导弹尾迹烟雾
  - 爆炸火球 + 碎片
  - 击中火花
  - Boost 时增强尾焰
- **相机效果**：
  - 击杀时短暂慢动作（0.5 秒）
  - Boost 时 FOV 扩大（速度感）
  - 被击中时屏幕闪红 + 轻微震动

### 6.2 音效

- 引擎声（随油门变化）
- 机枪射击声
- 导弹发射声 + 飞行声
- 锁定提示音（"嘀嘀嘀...嘀——"）
- 被锁定警告音
- 爆炸声
- 背景音乐（紧张的电子/摇滚风）

## 7. 技术架构

### 7.1 技术栈

| 层面 | 选择 |
|---|---|
| **3D 引擎** | Three.js |
| **构建工具** | Vite |
| **语言** | JavaScript (ES Module) |
| **3D 模型** | 低多边形自建（BufferGeometry） |
| **物理模拟** | 自定义简化飞行力学 |
| **音频** | Web Audio API |
| **输入** | Keyboard + Mouse + Touch |
| **画质管理** | Low / Medium / High 三档 |

### 7.3 游戏循环

```
每帧执行:
1. InputSystem.update()      → 收集输入
2. FlightPhysics.update()    → 更新飞行力学
3. WeaponSystem.update()     → 更新武器状态
4. LockOnSystem.update()     → 更新锁定状态
5. EnemyAI.update()          → AI 决策和行动
6. DamageSystem.update()     → 碰撞检测 & 伤害
7. EnergySystem.update()     → 能量恢复
8. ParticleSystem.update()   → 粒子效果
9. HUD.update()              → 更新 UI
10. Renderer.render()        → 渲染画面
```
